# 《晚唐风云》UI 美术升级方案 v1

更新时间：2026-04-12
定位：参考方案 / 执行草案

---

## 1. 目标

当前项目的核心问题，不是“单个面板不好看”，而是整套 UI 还停留在功能原型阶段：

- 地图、面板、弹窗、按钮没有形成统一世界观。
- 配色仍偏通用深色游戏面板，晚唐气质不够强。
- 组件样式过于同质，信息层级不清。
- emoji 图标严重削弱整体质感。
- 地图视觉与 UI 视觉尚未形成同一套材质语言。

本次升级的目标不是单纯“换皮”，而是建立一套能覆盖全项目的 UI 美术体系，使玩家第一眼就感受到：

- 这是晚唐，不是通用中世纪奇幻。
- 这是权力、军政、案牍、舆图构成的世界。
- 界面既有历史感，也有策略游戏应有的清晰度和可读性。

---

## 2. 总体风格方向

建议采用以下混合风格：

**晚唐舆图 + 官府文书 + 漆木金石**

拆解如下：

- 地图区：参考古地图、舆图、地志、行军图，不做现代 GIS 风格。
- 系统面板：参考案牍、公文、诏令、兵部文卷、节镇文书。
- 交互控件：参考木框、铜饰、印鉴、令牌、匣盒，而不是现代后台按钮。
- 数值展示：保持策略游戏的清晰度，但包装为“制度化信息”而非纯表格。

应避免的方向：

- 避免继续使用“深蓝+圆角卡片”的通用游戏原型风格。
- 避免纯欧式 CK3 复刻，否则会削弱题材辨识度。
- 避免大面积花纹堆叠，导致信息阅读困难。
- 避免高饱和、多彩、糖果色式势力配色。

---

## 3. 视觉原则

### 3.1 先气质，后细节

优先统一以下三件事：

- 主材质是什么
- 主色系是什么
- 面板像什么

在这三件事没统一前，不建议沉迷于局部按钮、阴影、特效微调。

### 3.2 信息层级必须强于装饰

策略游戏的 UI 首先是信息系统，任何装饰都不能压过：

- 当前关注对象
- 主要数值
- 当前可执行动作
- 当前风险/机会提示

### 3.3 地图与 UI 必须属于同一个世界

地图如果像古地图，面板就不能像现代控制台。
面板如果像古籍文书，地图也不能像纯几何示意图。

---

## 4. 色彩方案

建议建立三组颜色体系：基础色、功能色、势力色。

### 4.1 基础色

建议替换当前偏冷蓝的基础背景，改为更有材质感的低饱和色系：

- 主背景：黯墨黑 `#12100e`
- 面板底：乌木棕 `#1c1713`
- 次级底：漆青黑 `#211f22`
- 浮层底：烟墨灰 `#2a2520`
- 主文字：旧绢白 `#ddd2bf`
- 次文字：灰褐 `#9d917d`
- 边框：暗铜褐 `#4a3e31`

### 4.2 强调色

- 金色强调：泥金 `#b89a53`
- 红色强调：朱砂 `#8f3b2f`
- 绿色强调：铜绿 `#5d7a5a`
- 蓝色强调：石青 `#58718a`

建议用途：

- 金色：标题、主操作、焦点对象、玩家势力
- 朱砂：危险、拒绝、战事、紧急提醒
- 铜绿：正向增长、可执行、稳定状态
- 石青：中性信息、次级强调、水系相关信息

### 4.3 势力色

地图势力色建议遵循：

- 低饱和
- 偏矿物颜料和染料感
- 在纸面或地形底图上仍可读

推荐思路：

- 不直接大面积使用纯亮黄、纯亮紫、纯亮绿。
- 让政治色更像“罩染”，不是整块纯色填充。
- 玩家势力可独享较亮金色或暖金边界，以保证识别。

## 5. 字体体系

当前中文衬线方向是对的，但还需要明确分工。

建议建立四层字体角色：

### 5.1 标题字体

用途：

- 面板标题
- 人名主标题
- 大型章节标题

建议风格：

- 中文衬线
- 笔意略强
- 不要过于书法化，避免阅读成本太高

### 5.2 正文字体

用途：

- 描述文本
- tooltip
- 历史叙述

要求：

- 清晰
- 行距舒适
- 小字号仍可读

### 5.3 标签字体

用途：

- tab
- 小标题
- 数值标签
- 类别标签

要求：

- 规整
- 稍紧凑
- 与正文拉开气质差异

### 5.4 数字字体

用途：

- 顶部资源
- 战争数字
- 兵力、收入、时间

要求：

- 对齐稳定
- 宽度一致
- 在复杂背景上也容易读

建议：数字可单独使用一套更硬朗、更规则的西文字体，以增强“制度感”和“盘面感”。

---

## 6. 图标体系

这是当前最该优先替换的一项。

### 6.1 问题

当前大量 emoji 图标会带来三个问题：

- 气质不统一
- 视觉密度不稳定
- 高级感直接下降

### 6.2 建议方向

统一制作一套单色或双色 SVG 图标，主题偏：

- 官印
- 竹简
- 城池
- 甲兵
- 粮仓
- 钱帛
- 诏令
- 谋略
- 节钺
- 家族谱牒

### 6.3 图标风格要求

- 线条简洁，不要现代扁平办公风
- 不走过度写实路线
- 尽量保留“器物轮廓感”
- 小尺寸下优先轮廓可读性

---

## 7. 材质与装饰语言

建议克制使用真实材质，而不是铺满花纹。

### 7.1 推荐材质

- 旧纸纤维
- 漆木暗纹
- 铜边磨损
- 印泥红章
- 水墨式分割线

### 7.2 使用原则

- 大背景只用极弱纹理，避免抢字
- 标题栏可以增强材质感
- 主面板边框允许更明显的工艺感
- tooltip 和高频浮层要更轻，不能过重

### 7.3 不建议

- 不建议大面积金边描满所有组件
- 不建议所有面板都套卷轴边框
- 不建议贴图纹理过强导致文本发灰

---

## 8. 组件级升级建议

### 8.1 顶部资源栏

当前问题：

- 信息有用，但结构像开发期状态栏
- 每项权重接近，没有“仪表条”气质

建议：

- 改成“横向条案”或“诏册栏”结构
- 左右分组，主资源和次资源分开
- 资源名称弱化，数值强化
- 月变化采用更细腻的颜色和箭头/纹样提示
- tooltip 改成文书式信息卡

优先级：极高

### 8.2 右侧系统菜单

当前问题：

- 入口像普通工具栏按钮
- 图标辨识度低，文化气质弱

建议：

- 重做为“官署目录牌”或“悬挂牌”风格
- 图标统一换成自定义 SVG
- 当前选中状态做成压印/亮边/金线标记
- 不同系统入口可拥有轻微的职能差异感

优先级：极高

### 8.3 左侧人物面板

当前问题：

- 信息很多，但像数据堆栈
- 人物权力感、身份感、剧情感不足

建议：

- 顶部做成“人物档案抬头”
- 强化姓名、官职、谱系、所属势力
- 能力值区域做成独立主模块
- 特质与关系区分视觉层级
- 操作按钮从普通小图标升级成“文书命令/人物交互”入口

优先级：极高

### 8.4 通用弹窗

当前问题：

- 和普通面板差异不够
- 没有“事件发生”或“批复文书”的仪式感

建议：

- 区分“系统弹窗”“决策弹窗”“事件弹窗”三种样式
- 决策/事件弹窗允许更强的装饰头部
- 审批流和列表流则保持理性克制

优先级：高

### 8.5 底部时间栏

当前问题：

- 已有一些特色，但还不够统一

建议：

- 继续强化“历法 / 刻漏 / 朝廷时间装置”感
- 时间、年号、速度控制形成一个完整模块
- 速度按钮不要像普通播放器，做成制度化组件

优先级：高

### 8.6 Alert / Toast / WarOverlay

建议：

- AlertBar 更像“朝报”或“急递”
- EventToast 更像“飞报”或“邸报摘要”
- WarOverlay 更像“战局简报牌”

优先级：中高

## 9. 面板分型规范

建议把全项目 UI 面板统一分成以下四类。

### 9.1 主面板

适用：

- CharacterPanel
- TerritoryPanel
- GovernmentPanel
- RealmPanel

特征：

- 厚重
- 头部明确
- 允许装饰性边框
- 信息模块分段明显

### 9.2 次级面板

适用：

- 流程面板
- 列表面板
- 选择面板

特征：

- 更克制
- 装饰少
- 重点突出可读性与扫描效率

### 9.3 浮层

适用：

- tooltip
- toast
- 小型悬浮框

特征：

- 轻
- 快速
- 阴影浅
- 不能抢戏

### 9.4 仪表条

适用：

- 资源栏
- 时间栏
- 战争概览条

特征：

- 横向结构
- 强调节奏和状态变化
- 数值优先于装饰

---

## 10. 与地图美术的关系

UI 升级必须和地图升级使用同一世界观，但不必等待新版地图全部完成后才启动。

建议同步原则：

- 地图偏“舆图 / 地志 / 行军图”
- UI 偏“官文书 / 军报 / 案牍”
- 两者共用同一组材质、配色、边框语言

建议分开执行：

- 现在先做通用 UI 视觉系统
- 地图密度强相关部分，在 `100州 + 30道` 骨架稳定后再全量落地

---

## 11. 实施顺序

### 阶段 A：建立设计系统

交付物：

- 全局颜色 token
- 字体 token
- 图标风格说明
- 面板 / 按钮 / 标签 / tooltip 规范

目标：

- 先统一规则，再改页面

### 阶段 B：重做五个最高频区域

建议优先级：

1. 顶部资源栏
2. 右侧系统菜单
3. 左侧人物面板
4. 通用弹窗
5. 底部时间栏

目标：

- 让用户第一眼就感到“整个项目变了”

### 阶段 C：流程与系统面板统一

适用：

- 任命
- 调兵
- 谋略
- 决议
- 审批
- 战争流程

目标：

- 从“一个个单独修”转为“同一套系统皮肤”

### 阶段 D：地图区整体统一

适用：

- 地图边界
- 标签
- 地形底图
- 战争与行军覆盖层

前提：

- `100州 + 30道` 骨架基本稳定

---

## 12. 近期建议的具体动作

如果只做最有性价比的一轮升级，建议按以下顺序推进：

1. 先替换 emoji 图标，建立 SVG 图标系统。
2. 重做全局颜色变量，摆脱当前“深蓝原型风”。
3. 重构 `Button / Modal / Panel / Tooltip` 基础组件。
4. 优先重做 `ResourceBar / SideMenu / CharacterPanel / TimeControl`。
5. 选 1 个高频流程面板做完整样板，作为后续模板。

---

## 13. 不建议的做法

- 不建议先逐页零散美化，容易越改越乱。
- 不建议地图和 UI 各做一套风格。
- 不建议继续依赖 emoji 作为正式图标系统。
- 不建议过度追求“古风装饰”，牺牲可读性。
- 不建议在 `100州 + 30道` 结构尚未稳定前，重做地图区全部排版。

---

## 14. 结论

《晚唐风云》的 UI 升级，本质上不是一次局部修饰，而是一次“视觉世界观建立”。

最优路线是：

- 先建立统一的 UI 美术体系
- 再优先重做高频核心区域
- 地图相关重视觉在新版地图骨架稳定后统一落地

最终目标不是“像 CK3”，而是做到：

**一眼看上去就知道这是《晚唐风云》，而且它有自己的时代气质、制度气质与地图气质。**
