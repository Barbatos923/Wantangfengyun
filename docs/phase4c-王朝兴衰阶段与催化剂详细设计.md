# Phase 4C 王朝兴衰阶段与催化剂详细设计

本文档基于《晚唐风云》现有的纯函数引擎架构（Zustand Stores + 月结 Pipeline），详细定义了“王朝兴衰”状态机的三个核心阶段（治世、危世、乱世）的机制差异，并系统梳理了驱动状态流转的“催化剂（Catalysts）”行动清单。

---

## 一、 核心状态机与阶段差异

王朝的局势由全局唯一的**天命值（Mandate，范围 0~200）**驱动。天命值不仅是一个表现数值，更是底层游戏规则的“开关”。

### 1. 治世（盛世） | 天命值 150 ~ 200
**核心特征**：中央集权稳固，政通人和，国家机器高效运转。
* **经济规则**：全国税收上缴率下限强制提高（如最低 50%），基础农业与商业产出获得 +20% 时代 Buff。
* **军事规则**：内部和平。发动“武力兼并”宣战的成本极高（威望 -40，正统 -20，*已在 `warCalc.ts` 实现*），极大限制了节度使的内部扩张。
* **人事规则**：中央（皇帝/吏部）拥有绝对权威。可以无惩罚地驳回或收回地方节度使的“辟署权”；流官考课的“贤能”积累速度 +50%。
* **稳定规则**：叛乱阈值极高，军队士气自然衰减速度减缓。

### 2. 危世（衰退） | 天命值 50 ~ 149
**核心特征**：藩镇割据抬头，朝堂党争激烈，中央对地方控制力显著下降。（游戏开局默认状态）
* **经济规则**：地方财政自主。节度使可以合法截留税收（上缴率可调至 10% 甚至 0 而不受严惩），导致中央财政吃紧。
* **军事规则**：武力兼并成本降低（威望 -20，正统 -10，*已实现*）。节度使之间可以相对自由地互相宣战、吞并。
* **人事规则**：节度使获得事实上的“宗法继承权”和完全的“辟署权”。中央若强行干预地方人事，将引发巨大的好感惩罚，甚至直接导致叛乱。
* **稳定规则**：军队士气衰减加快，兵变概率回归正常水平。

### 3. 乱世（崩溃） | 天命值 0 ~ 49
**核心特征**：天下大乱，皇权旁落，群雄逐鹿。
* **经济规则**：全国经济遭受毁灭性打击，产出 -50%，人口增长停滞或负增长。
* **军事规则**：解锁“争夺天下”宣战理由，武力兼并几乎无成本（威望 -5，正统 0，*已实现*）。围城进度加快，军队补给消耗降低，鼓励全面战争。
* **人事规则**：中央官职系统瘫痪。所有领地强制转为“军管”状态，继承法全部退化为“宗法继承”或“武力夺权”。
* **稳定规则**：天命值极低且难以自然恢复。**唯一出口**：某位势力领主控制超过半数以上的道级行政区，并举行“登基大典”，方可一次性获得大量天命值，终结乱世，重建治世。

## 二、 催化剂（Catalysts）：驱动兴衰的行动清单

催化剂是导致天命值增减的具体游戏行为。根据对现有引擎代码（`warSystem.ts`, `militarySystem.ts`, `economySystem.ts`, `appointAction.ts` 等）的盘点，我们将催化剂分为**已具备触发条件（可直接挂载）**和**待后续构建**两类。

### 1. 已有系统可直接挂载的催化剂

现有代码已经产生了丰富的状态和事件，只需在 `TurnManager` 的月结回调中扫描这些状态即可触发天命值变化，**无需修改原有业务逻辑**。

| 催化剂名称 | 触发条件与代码位置 | 天命值影响 | 逻辑说明 |
| :--- | :--- | :---: | :--- |
| **中央平叛/拓边胜利** | `warSystem.ts` (`settleWar`) | **+5** | 皇帝或中央军作为攻方/守方赢得战争。 |
| **中央战败/丧失领土** | `warSystem.ts` (`settleWar`) | **-10** | 皇帝或中央军战败，或京畿道领土被占领。 |
| **藩镇私自兼并** | `warSystem.ts` (`settleWar`) | **-5** | 两个非皇帝势力的节度使之间完成领土吞并。 |
| **地方兵变** | `militarySystem.ts` (兵变检查) | **-3** | 军队士气过低触发兵变（通过拦截 `GameEvent` 识别）。 |
| **重镇陷落** | `warSystem.ts` (围城结算) | **-5** | 州级治所或重要关隘被攻破（拦截 `城破` 事件）。 |
| **中央财政破产** | `economySystem.ts` (破产检查) | **-10** | 皇帝角色的 `money` 或 `grain` < -50000。 |
| **皇帝非正常死亡** | `characterSystem.ts` (死亡结算) | **-20** | 皇帝驾崩，尤其是遇刺或无嗣而终。 |
| **强行罢免引发众怒** | `appointAction.ts` (`executeAppoint`) | **-2** | 罢免高品级官员导致单次好感度惩罚超过 -30。 |

### 2. 待后续构建的催化剂

以下催化剂需要配合 Phase 4C 及后续阶段的新功能（如派系系统、随机事件、高级交互）来实现。

| 催化剂名称 | 依赖的新系统/交互 | 天命值影响 | 逻辑说明 |
| :--- | :--- | :---: | :--- |
| **截留赋税** | 经济上缴率调整 UI | **-2 / 月** | 节度使将上缴率设定低于时代要求，且中央无力惩罚。 |
| **开科取士** | 举办科举交互 | **+10** | 皇帝或宰相成功举办科举，录用高贤能人才。 |
| **考课败坏** | 考课统计系统 | **-5 / 次** | 三年一考结算时，全国“下等”评价官员比例过高。 |
| **天灾/祥瑞** | 随机事件系统 | **-10 / +5** | 发生大旱、蝗灾、黄河泛滥（减分）；发现祥瑞（加分）。 |
| **党争倾轧** | 派系系统 | **-5** | 朝廷重要岗位（如六部尚书）因党争长期空缺，或宰相疯狂排斥异己。 |
| **僭越称王** | 僭越/称帝交互 | **-30** | 节度使自立为王，公然挑战中央正统（直接将局势推向乱世）。 |
| **天命归一** | 登基大典交互 | **+100** | 乱世中，控制足够领土的霸主正式称帝，终结乱世。 |

## 三、 架构实现建议

为了保持引擎的纯函数特性和模块解耦，建议采用以下架构实现催化剂系统：

1. **新增 `DynastyStore`**：创建一个独立的 Zustand Store，专门存储 `mandate`（天命值）和 `currentEra`（当前时代）。
2. **集中结算 (`catalystSystem.ts`)**：在 `settlement.ts` 的末尾新增一个 `runCatalystSystem(date)`。该函数负责：
   - 读取本月新增的 `GameEvent`（如兵变、城破）。
   - 读取 `WarStore`、`CharacterStore` 的最新状态（如破产、战争结束）。
   - 根据上述信息，匹配催化剂清单，计算本月天命值的总变化量。
   - 更新 `DynastyStore`，若天命值跨越阈值，则触发时代切换，并生成“时代变迁”的重大历史事件。
