# 《晚唐风云》Phase 6：NPC 行为模块设计方案

本文档基于 Phase 6 的 NPC Engine 架构（`NpcContext` + `PlayerTask`），详细设计目前游戏中**玩家可操作但 NPC 尚未实现**的核心行为模块。这些模块是让游戏世界真正"活起来"的关键。

---

## 一、最紧迫的系统级修复：战争自动结算

在设计 NPC 行为之前，必须先修复当前战争系统的一个致命缺陷：**战争分数到达 ±100 时不会自动结束，只能由玩家手动点击和谈**。这会导致 NPC 之间的战争永远卡死。

### 修复方案（`warSystem.ts`）

在 `runWarSystem` 的月结逻辑末尾，增加自动结算检查：

```typescript
// engine/systems/warSystem.ts
import { settleWar } from '@engine/military/warSettlement';

export function runWarSystem(date: GameDate): void {
  // ... 现有的行军、战斗、围城逻辑 ...

  // ===== 战争自动结算检查 =====
  for (const war of useWarStore.getState().wars.values()) {
    if (war.status !== 'active') continue;

    if (war.warScore >= 100) {
      // 攻方绝对优势，强制防方投降
      settleWar(war.id, 'attackerWin');
    } else if (war.warScore <= -100) {
      // 防方绝对优势，强制攻方投降
      settleWar(war.id, 'defenderWin');
    }
  }
}
```

## 二、核心行为模块设计

以下行为模块需要按照 Phase 6 的 `NpcBehavior` 接口规范进行实现。

### 1. 宣战行为（`declareWarBehavior`）

**目标**：NPC 统治者根据自身性格和实力，主动向周边势力发动战争。

- **触发者**：所有统治者（`isRuler === true`）。
- **前置条件**：
  - 当前未处于战争状态（避免多线作战）。
  - 拥有至少一支军队。
  - 存在合法的宣战理由（相邻领地可兼并，或有法理宣称，或是附庸谋求独立）。
- **意愿权重（Weight）计算**：
  - **基础分**：0
  - **性格修正**：`boldness`（胆识）每点 +2，`rationality`（理性）每点 -1，`greed`（贪婪）每点 +1。
  - **实力对比**：计算双方总兵力。如果己方兵力 > 敌方 1.5 倍，权重 +50；如果 < 敌方，权重 -100（理性高的 NPC 绝不打无准备之仗，胆识极高的可能冒险）。
  - **好感度修正**：对目标好感度越低，宣战意愿越高（好感 < -50 时权重 +30）。
- **NPC 执行逻辑（`executeAsNpc`）**：
  - 选定权重最高的目标和理由，调用 `executeDeclareWar`。
  - **关键联动**：宣战后，必须立即触发"军事动员"逻辑（见下文），否则宣战无意义。
- **玩家模式（`playerMode`）**：`skip`（玩家主动在 UI 宣战，引擎不推送任务）。

### 2. 军事动员行为（`mobilizeBehavior`）

**目标**：NPC 在处于战争状态时，自动组建行营并派兵出征。这是宣战的配套行为。

- **触发者**：处于战争状态的统治者。
- **前置条件**：
  - 处于战争状态。
  - 尚未组建针对该战争的行营。
- **意愿权重**：`Infinity`（强制触发，无视 `maxActions` 限制）。只要在打仗且没发兵，就必须发兵。
- **NPC 执行逻辑**：
  1. 找出自己所有的军队。
  2. 调用 `executeCreateCampaign` 组建行营，任命军事能力最高的将领为都统。
  3. 确定行军目标：
     - 如果是攻方：选择敌方距离最近的领地（或战争目标领地）。
     - 如果是防方：选择己方被围攻的领地，或敌方行营所在的领地进行拦截。
  4. 调用 `executeSetCampaignTarget` 设定路线。
- **玩家模式**：`skip`（玩家在 UI 手动组建行营）。

### 3. 征兵与补员行为（`recruitBehavior`）

**目标**：NPC 在兵力不足时，自动消耗人口和资金补充军队。

- **触发者**：所有统治者。
- **前置条件**：
  - 领地内有足够的征兵池（`conscriptionPool`）和资金。
  - 现有军队未满编（需要补员），或总兵力低于领地承载上限（需要新建营）。
- **意愿权重**：
  - **基础分**：根据兵力缺口比例计算（缺口越大，权重越高）。
  - **环境修正**：处于战争状态时，权重 +50。处于乱世时，权重 +20。
  - **性格修正**：`boldness` +1，`greed` -1（贪财的 NPC 不愿花钱养兵）。
- **NPC 执行逻辑**：
  - 优先调用 `executeReplenish` 为现有不满编的营补员。
  - 如果资金充裕且需要扩军，调用 `executeRecruit` 创建新营。
- **玩家模式**：`skip`。

### 4. 要求效忠行为（`demandFealtyBehavior`）

**目标**：NPC 统治者通过外交手段（而非战争）扩张势力，要求弱小邻居臣服。

- **触发者**：所有统治者。
- **前置条件**：
  - 目标无主岗位，或目标的主岗位在自己法理管辖范围内。
  - 己方军力显著大于目标。
- **意愿权重**：
  - **基础分**：0
  - **性格修正**：`greed` +2，`boldness` +1，`honor` -1。
  - **实力对比**：军力碾压越严重，权重越高。
- **NPC 执行逻辑**：
  - 调用 `executeDemandFealty`。
- **玩家模式**：`skip`。

### 5. 建设行为（`buildBehavior`）

**目标**：NPC 在和平时期，利用多余资金发展领地经济和防御。

- **触发者**：所有统治者。
- **前置条件**：
  - 处于和平状态（战争期间不搞建设）。
  - 资金和粮食充裕（大于某个安全阈值）。
  - 领地内有空闲建筑槽位，或有可升级的建筑。
- **意愿权重**：
  - **基础分**：资金越充裕，权重越高。
  - **性格修正**：`administration`（内政能力）每点 +2，`rationality` +1。
- **NPC 执行逻辑**：
  - 随机或根据需求（缺钱建集市，缺粮建农田，缺兵建兵营）选择一个建筑。
  - 调用 `executeBuild`。
- **玩家模式**：`skip`。

### 6. 赏赐行为（`rewardBehavior`）

**目标**：NPC 在军队士气低落时，花钱稳定军心，防止兵变。

- **触发者**：所有拥有军队的角色。
- **前置条件**：
  - 麾下有军队的平均士气低于 40。
  - 拥有足够的资金。
- **意愿权重**：
  - **基础分**：士气越低，权重越高（士气 < 20 时权重极大，因为面临兵变风险）。
  - **性格修正**：`greed` -2（贪财的 NPC 宁可兵变也不发钱），`rationality` +2。
- **NPC 执行逻辑**：
  - 调用 `executeReward`，消耗资金提升士气。
- **玩家模式**：`skip`。

## 三、实施建议

这些行为模块的实现可以分批进行，建议的优先级如下：

1. **P0（阻断性修复）**：`warSystem.ts` 中的战争自动结算。
2. **P1（战争闭环）**：`declareWarBehavior` 和 `mobilizeBehavior`。这两个必须同时上，否则 NPC 战争系统无法运转。
3. **P2（军力维持）**：`recruitBehavior` 和 `rewardBehavior`。保证 NPC 有兵可打，且不会轻易兵变。
4. **P3（日常发展）**：`buildBehavior` 和 `demandFealtyBehavior`。丰富和平时期的世界动态。

所有这些行为都应遵循 Phase 6 设计的 `NpcBehavior` 接口，统一注册到 NpcEngine 的调度循环中。
