# Phase 4C — 王朝兴衰机制：系统设计方案

> 版本：v1.1 | 日期：2026-04-01
> 基于 manus 初版设计 + Claude Code 审阅反馈 + 用户确认的最终方案。

---

## 一、核心设计理念："唯器与名，不可以假人"

在《晚唐风云》中，**正统性（Legitimacy）**不再是一个空洞的数值或仅靠仪式获取的资源。天朝的正统性深深植根于其复杂的**官职本位体系**中。正统性代表了"名器"，是国家机器对个人地位的背书。

正统性是一种**极难获取、极易流失、且至关重要**的战略资源。它与角色的品位、职位深度绑定，形成一套严密的逻辑闭环。

---

## 二、正统性的生成机制（与官职系统挂钩）

正统性的数值框架（0~100）由以下维度构建：

### 2.1 品位（Rank）：决定正统性的"上限"

品位代表了角色在官僚体系中的绝对阶级，决定了其正统性的**最大上限（Cap）**。

| 品级 | Rank Level | 上限 |
|------|-----------|------|
| 九品 | 1~4 | 40 |
| 八品 | 5~8 | 50 |
| 七品 | 9~12 | 60 |
| 六品 | 13~16 | 70 |
| 五品 | 17~20 | 80 |
| 四品 | 21~24 | 85 |
| 三品 | 25~26 | 90 |
| 二品 | 27~28 | 95 |
| 一品 | 29 | 100 |

**软限制设计**：品位不足不再硬性阻止任命，而是通过正统性 Cap 低于岗位预期 → 好感 debuff 来产生代价。铨选中低品位候选人仍然出现，但评分大幅减值并标注警告。NPC 不会选择品位不足的候选人。

### 2.2 职位（Position）：提供正统性的"保底刷新"

每个岗位有一个**基础正统性（baseLegitimacy）**：

| 岗位类型 | baseLegitimacy |
|---------|---------------|
| 皇帝（pos-emperor） | 95 |
| 五品及以上（minRank ≥ 17） | 80 |
| 六品及以下（minRank < 17） | 60 |

**机制**：当角色**获得该岗位时**，如果其当前正统性低于该岗位的基础值，则**瞬间刷新**至该基础值（受品位 Cap 约束）。

**预期正统性 E**：取角色当前持有的所有岗位中最高的 baseLegitimacy。无岗位角色正统性不生效（显示 NA）。

### 2.3 爵位与勋阶（待建）

- **爵位（Nobility）**：提供正统性的"月度增益"（如县男 +0.5/月，国公 +2/月）。
- **勋阶（Honor）**：提供正统性的"一次性爆发"（如获封上柱国 +30）。

## 三、时代衰减

正统性在不同的时代背景下，其维持成本截然不同。

### 3.1 时代对皇帝正统性的影响

| 时代 | 皇帝月度变化 | 说明 |
|------|-------------|------|
| 治世 | 0 | 天下承平，名器稳固 |
| 危世 | -0.25/月 | 约每 4 个月 -1，10 年约 -30 |
| 乱世 | -1/月 | 约 3 年吃满好感 debuff |

### 3.2 正统性对好感度的传导

正统性的核心价值：影响**所有人**好感度的通用全局 Buff/Debuff。由两部分叠加构成：

**第一部分：预期差值**

设 B 的当前正统性为 `L`，B 当前最高岗位的预期正统性为 `E`，差值 `D = L - E`：

| 差值 D | 好感度修正 | 含义 |
|--------|-----------|------|
| D ≥ +10 | **+10** | 远超预期 |
| 0 ≤ D < +10 | **0** | 名副其实 |
| -10 ≤ D < 0 | **-5** | 略显不足 |
| -20 ≤ D < -10 | **-15** | 人心浮动 |
| -30 ≤ D < -20 | **-30** | 离心离德 |
| D < -30 | **-50** | 众叛亲离 |

**第二部分：数值影响**

| 条件 | 好感度修正 | 标签 |
|------|-----------|------|
| L ≥ 90 | **+10** | 天命所归 |
| L ≤ 30 | **-20** | 名器尽失 |

两部分叠加生效。极端情况下（如皇帝从 95 跌至 25），可达 -70 的好感惩罚，此时天下应当群起叛乱。

**传导逻辑闭环**：
危世降临 → 时代 Debuff 导致正统性每月持续流失 → 正统性跌破预期 → 触发全局好感度惩罚 → 封臣离心，拒绝上缴赋税，甚至发动叛乱 → 叛乱和财政危机进一步推动"危世→乱世"的进度条。

## 四、天朝时代

时代切换基于**进度条**的诱因驱动型状态机。

### 4.1 时代状态

三种时代：**治世**、**危世**、**乱世**。每种时代影响：
- 皇帝正统性自然变化（见 3.1）
- 武力兼并宣战理由的正统性花费（已有实现）

### 4.2 进度条机制

- **稳定进度（stabilityProgress）**：0~100，达到 100 时代向好切换
- **崩溃进度（collapseProgress）**：0~100，达到 100 时代向坏切换

切换时两个进度条归零。

### 4.3 本阶段实现的诱因

**治世 → 危世（衰退进度条）**：慢性侵蚀驱动
- 土地兼并：每年自动 +1
- 皇帝正统性低于预期：每年 +5

**危世 → 乱世（崩溃进度条）**：急性冲击驱动
- 皇帝正统性低于预期：每年 +5
- 针对皇帝的独立战争胜利：每场 +10

**恢复方向（本阶段不实现）**：
- 危世 → 治世（中兴）：留待 Phase 6 政治/派系玩法
- 乱世 → 治世（一统）：涉及皇帝岗位创建/销毁，留待后续

## 五、初始化规则

| 角色类型 | 初始正统性 |
|---------|-----------|
| 皇帝 | 85（手动配置） |
| 有官者 | 最高岗位的 baseLegitimacy |
| 无官者 | 30 |

继承时正统性**不继承**，靠继承岗位的 baseLegitimacy 刷新。

## 六、与宣战系统的交互

宣战花费正统性的机制已实现（`warCalc.ts`），按时代分档。本阶段补充：
- **资源校验**：宣战前验证 prestige/legitimacy 足够，不足则不允许宣战
- **独立战争胜利**：如果是针对皇帝的独立战争胜利，推动崩溃进度 +10

## 七、本阶段不实现

- 皇帝岗位销毁/创建（乱世↔治世切换触发器）
- 稳定方向的恢复诱因（Phase 6）
- 爵位/勋阶系统
- 非宗法皇位更替（禅让/篡位）
