## 1. 游戏概述与核心命题

### 1.1 一句话定义

《晚唐风云》是一款**以角色为驱动的中国历史模拟游戏**，玩家扮演晚唐时代的一位角色，通过政治、军事与外交手段在乱世中生存、扩张，体验晚唐时代的生活，并最终将自己的一生以 AI 生成的史书形式留存下来。是再造大唐，还是在乱世偏安一隅，抑或是作为侠客游侠四方？我们希望能够给玩家最原汁原味的中国古代角色扮演历史模拟体验。

### 1.2 游戏的核心命题

本游戏要回答一个问题：**如果你是晚唐的一位人物，你会如何度过自己的一生？”**

譬如**如果你是一位被牙兵拥立的节度使，你会如何在"被朝廷控制"与"被自己的士兵推翻"这两种死法之间活下去？**

这个问题没有标准答案。每一次决策都是两害相权取其轻。游戏的乐趣不在于"找到最优解"，而在于**在不断的危机中做出选择，并承担后果**。

### 1.3 游戏的独特卖点（USP）

本游戏有两个核心卖点，缺一不可：

**卖点一：真实的中国式权力博弈。** 市面上的历史策略游戏要么是欧洲封建制（骑士、教皇、圣战），要么是架空策略（文明、帝国时代）。没有一款游戏以模拟了中国历史为最早优先级。我们的第一个剧本选择在晚唐时期，希望尽量模拟出当时藩镇割据、牙兵逐帅、宦官专权、科举与士族并举的局面。本游戏将这些机制作为核心玩法，而非装饰。

**卖点二：AI 自动生成专属史书。** 玩家的每一次游戏都会生成一本独一无二的《某某书》，以文言文纪传体记录玩家角色的一生。这本史书可以导出、分享，是玩家在游戏中最终的成就凭证。

## 2. 玩家体验目标

在开始设计任何系统之前，必须明确我们希望玩家在游戏中产生的情感体验。以下三种体验是所有系统设计的最终目标：

### 2.1 “人物扮演”的沉浸感

我们不希望玩家如同玩文明或欧陆风云一样，只是追求数值的最大化。我们希望玩家能够沉浸式地扮演晚唐人物，通过面临一个又一个的选择，来诠释和扮演人物。譬如作为节度使，朝廷要你进贡，牙兵要你发赏赐，世家要你给他们的子弟官职，清流要你推行德政。钱只有这么多，你必须选择先满足谁。是一次次的行动和选择塑造了人物。

### 2.2 "人心难测"的戏剧感

游戏中的每一个 NPC 都有自己的欲望和底线。你最信任的大将可能因为你一次不公平的赏赐而心生怨恨；你最讨厌的政敌可能在你最危难的时候伸出援手。游戏要让玩家感受到：**人不是工具，而是有自己逻辑的独立个体**。

### 2.3 "青史留名"的成就感

玩家在游戏中做的每一个行动和选择，最终都会被 AI 史官以优美的文言文记录下来。一个玩家打了一辈子仗，他的史书里就是"百战之将，威震殊俗"；一个玩家专注内政，他的史书里就是"轻徭薄赋，百姓安居"。史书是玩家游戏行为的镜子，也是最终的成就感来源。

## 3. 核心玩法循环

### 3.1 宏观循环（一局游戏的结构）

一局游戏的时间跨度约为 400-500 游戏年，从晚唐开始，到蒙古入侵作为最后的大boss收关。玩家会扮演同一家族的数代人。我们用一套天朝兴衰的循环机制来模拟中国的古代的王朝周期律，并定义游戏的不同阶段。其本质世一套基于状态机（State Machine）的全局环境系统。它通过定义不同的"时代（Era）"来改变游戏的全局规则，并通过"诱因（Catalysts）"来推动时代之间的转换，从而模拟中国历史上的治乱循环。

时代包括三种：
1. 危世：天命动摇，与农民起义概率大增。可能的buff(需要后续设计)：税收减少；腐败事件频发；民粹与农民起义概率大增；边疆地区倾向独立

2. 乱世：天下大乱，所有藩镇独立，群雄逐鹿。可能的buff(需要后续设计)：最高头衔（天朝）被销毁；所有藩镇独立；群雄逐鹿。

3. 治世：开疆拓土，抑或是追求四海升平。可能的buff(需要后续设计)：加速大型工程建设；科举产出高贤能文官；允许更大规模的常备军；解锁"都护府"行政路类型

时代之间的驱动力是诱因系统，时代之间的转换不是按固定时间发生的，而是由诱因（积分系统）驱动的。当某种诱因的积分积累到阈值时，就会触发时代转换。

诱因分为三种类型：1）立即触发型（致命打击）：如"正统性降至零"、"新家族篡位"，直接导致时代跌落。2）一次性事件型（行为惩罚）：如"皇帝战败"、"首都沦陷"、"皇族被暗杀"，每次发生增加一定的不稳定积分。3）持续状态型（结构性腐败）：如"国库空虚"、"长期未举办科举"、"任命低能官员"、"反对派势力过大"，每月/每年持续增加不稳定积分；4）根本因素：如“土地兼并”、“气候变化”等，作为治世无法永久的底层因素。

循环的闭环逻辑：标准的循环路径是：治世 → 危世 → （乱世） → 新的治世。在治世，皇帝的不当治理、天灾人祸、抑或是根本性的土地兼并将导致通向危世的积分不断累计，皇帝或臣子的励精图治可以延缓这一进程，让王朝更长期地处于治世，但终将进入危世。在危世中，如果没有英雄力挽狂澜，那么就会滑入乱世。在乱世，群雄割据，决出新的王朝，再造新朝盛世。

在晚唐，我们认为初始处于危世之中，玩家可以选择努力将危世拽回治世，也可以加速危世向乱世的滑落，并在乱世征服天下。

### 3.2 微观循环（每个游戏月的节奏）

每个游戏月，玩家需要处理以下事务：

**随机事件（系统自动触发）**：
- 我们希望制作多个不同主题的事件包，例如“战争”、“家庭教育”、“生活方式”、“旅行”等，在不同的条件下，可能触发不同的随机事件；

**主动行动（玩家主动发起）**：
- 外交；
- 人事；
- 工程；
- 军事；
- 活动；
- 计谋；
- 与其他角色的其他交互；

**每月结算**：
- 资源变动（钱粮 +/- ，各项指标 +/-）；
- 起居注在后台记录，并在年底自动生成当年的史书。

## 4. 世界观与历史背景

### 4.1 时代背景

游戏开始于 870 年前后的晚唐。这是一个皇权衰落的危世，南诏频繁入侵、河北藩镇割据、中央宦官专权，随后的庞勋、王仙芝、黄巢起义将帝国推向了彻底的深渊，大唐陷入藩镇割据的乱世之中。

这个时代有以下几个核心特征，游戏必须真实反映：

**特征一：藩镇割据。** 根据《唐代藩镇研究》当时的全国藩镇大体可以分为四类：河北割据型、中原防遏型、边防防遏型、东南忠顺型。在 870 年前后的时间点，除了河北的藩镇对大唐名义上称臣、实际上独立之外，全国绝大多数的节度使仍然由朝廷任命。但随着格局的推进，尤其是在黄巢起义开始前后，各地的藩镇就出现了强烈的离心倾向。等到黄巢攻入长安，皇帝被迫南迁，各地的节度使几乎就开始完全自立，朝廷能任命的节度使也就越来越少。

**特征二：牙兵逐帅。** 节度使的核心武装叫"牙兵"，是世代相传的职业军人。牙兵战斗力极强，但也极为骄横。历史上多次发生牙兵不满节度使的管理方式，直接将其杀掉、换一个新节度使的事件（史称"牙兵逐帅"）。这是晚唐政治最黑色幽默的一面：你的保镖随时可能成为你的刽子手。从研究中我们看到，衙兵的跋扈是不分对朝廷忠顺与否的，无论是河北藩镇还是除河北之外的其他藩镇，都多次出现了类似的情形。

**特征三：宦官专权。** 唐朝中后期，宦官掌握了禁军（皇帝的直属军队）的控制权，实际上控制了皇帝的人身安全。皇帝的废立，往往由宦官决定。朝臣与宦官之间往往有激烈的矛盾，皇帝则时而依赖宦官，时而又想除掉宦官。这一现象是朝廷政治最核心的矛盾。

**特征四：科举与世家的博弈。** 科举制度为寒门子弟提供了进入权力中枢的通道，但世家大族（如崔、王、李、郑等"五姓七望"）凭借几百年的积累，仍然在官场上占据主导地位。这两股力量的博弈，是朝廷政治的另一条主线。

### 4.2 地图设计

游戏地图覆盖唐朝全境（大致相当于今天的中国本土加上部分中亚、东南亚地区），分为以下几个区域：

| 区域 | 历史特征 | 游戏特点 |
| :--- | :--- | :--- |
| **关中（长安周边）** | 皇权所在地，朝廷直辖 | 合法性资源丰富，但受朝廷监控严格 |
| **河北（今华北平原）** | 最强大的藩镇集中地 | 军事资源丰富，但藩镇势力根深蒂固 |
| **江南（今长江以南）** | 经济最发达的地区 | 钱粮资源丰富，但军事力量薄弱 |
| **西北（今甘肃、宁夏）** | 吐蕃、回鹘等游牧民族的活动区域 | 可招募强力蕃兵，但边境不稳定 |
| **东北（今辽宁）** | 契丹崛起的区域 | 外患压力大，但可借助契丹力量打击对手 |
| **西南（今云贵川）** | 南诏入侵，地方割据 | 少数民族众多，南诏频繁入侵，相对忠顺朝廷 |

## 5. 角色系统

### 5.1 角色的基本构成

游戏中的每一个角色（包括玩家角色和所有 NPC）都由以下四个维度构成：

#### 维度一：五项基础能力（0-30 分制）

| 能力 | 含义 | 影响 |
| :--- | :--- | :--- |
| **军事** | 统兵打仗的能力 | 战斗力加成、关键战役成功率 |
| **管理** | 管理领地、收税的能力 | 税收效率、建设速度 |
| **谋略** | 阴谋、情报、暗杀的能力 | 阴谋成功率、反阴谋能力 |
| **外交** | 游说、谈判、联姻的能力 | 外交行动成功率、好感度加成 |
| **学识** | 儒家经典、文化修养 | 清流名望加成、科举录取质量 |

能力值在角色出生时随机生成，并受家族遗传（父母能力高的角色，子女能力也更可能偏高）和教育影响。能力值不会随时间自动提升，但可以通过特定事件（如"拜师学艺"、"战场历练"）提升。

#### 维度二：特质（Traits）

特质是影响角色行为逻辑和事件触发的核心机制，包括性格特质、教育特质、先天特质、其他特质。

1）先天特质，由基因决定，父母拥有某些先天特质，则子女有概率遗传先天特质。如果父母双方拥有相同的特质，遗传概率会大幅增加。如天才、倾国倾城、强壮。
2）性格特质，每个角色在教育过程中会逐步获得 2-4 个性格特质，如勇敢、勤勉、懒惰、多疑等，这些特质通常不会改变。
3）教育特质，角色成年时会根据其童年教育重心获得一个教育特质，对应军事、管理、谋略、外交、学识五个方向，根据教育的结果分为四级。等级越高，提供的属性加成和对应生活方式经验加成越高。
4）其他特质，角色的经历和特定事件会使得角色获得其他特质，如如"在战场上受到重伤"后获得"伤疤"特质。

#### 维度三：人际关系网络

每个角色与其他角色之间有一个"好感度"（-100 到 +100）。好感度影响：
- A对B的好感度越高，A 越容易做出对 B 有利的交互，包括：赠送礼物、结盟、联姻、支持战争等
- 反之，A 对 B 的好感度越低，A 越容易做出对 B 不利的交互，包括宣战、撕毁盟约、实施阴谋等。

**关系值的影响因素**：
- 一般好感度：影响所有角色对你的看法，受到外交能力、特定特质的影响；
- 吸引力好感度：受到先天特质、性格特质的影响，拥有相同特质的角色之间通常会有好感度加成；
- 前任好感度继承：当统治者死亡时，继承人会暂时继承封臣对前任统治者的部分好感度
- 人际关系影响：角色之间建立的特定关系会提供很高的好感度修正，如朋友、情侣、死敌
- 直接互动及事件影响（赏赐 → 关系上升，惩罚 → 关系下降；"你杀了他的父亲"会导致永久性的仇恨）；

#### 维度四：健康与寿命

每个角色都有健康值（0-100）。健康值受以下因素影响：
- 年龄（超过 50 岁后，健康值每年自然下降）；
- 压力（见下文）；
- 随机疾病事件；
- 战场受伤。

当健康值降至 0 时，角色死亡，触发继承系统。

### 5.2 压力系统（Stress System）

**压力系统**是防止玩家"无限最优化"的核心机制。

**基本逻辑**：每个角色都有一套"核心价值观"（由其性格特质决定）。当玩家做出违背角色价值观的决策时，角色会积累"压力"。

**示例**：
- 一个"仁慈"特质的节度使，下令屠杀战俘 → 压力 +30；
- 一个"廉洁"特质的节度使，接受了敌方的贿赂 → 压力 +20；
- 一个"骄傲"特质的节度使，被迫向敌人低头求和 → 压力 +25。

**压力的后果**：压力过大，会出现一系列debuff
- 压力 > 50：角色开始出现"烦躁"特质，外交能力 -3；
- 压力 > 75：角色可能触发"酗酒"或"沉迷声色"等负面事件，健康值开始下降；
- 压力 = 100：角色精神崩溃，可能猝死或做出极端行为。

**压力的释放**：可以通过多种活动或事件释放压力，例如
- 举办宴会（消耗钱粮，压力 -20）；
- 出行游猎（需要一段时间无法处理政务，压力 -30）；
- 与挚友深谈（需要有"挚友"关系的角色存在，压力 -25）。

## 6. 核心资源

游戏中有四种核心资源。**它们不可互相直接兑换**，每种资源都有其专属用途，重视不同的资源意味着不同的扮演体验。

#### 资源一：钱（政治与文治资本）

钱（铜钱/绢帛）代表了商业财富和货币储备，主要用于维持文官体系和外交活动，是消耗类资源。

**来源**：其主要来源是领地系统和官职系统，包括领地的商税、特定领地收入、朝廷俸禄。

**消耗**：其主要消耗在领地系统、军事系统、官职系统和活动系统，包括领地建设、军队招募和赏赐、下级官员的俸禄、各类活动如举办科举、宴会等。


#### 资源二：粮（军事与民生资本）

粮（军粮/粟米）代表了农业产出和生存物资，是维持军队战斗力的直接资源，是消耗类资源。

**来源**：主要来源是领地系统、外交系统和军事系统，包括领地农业税、外交购买贸易、战争劫掠；

**消耗**：主要消耗在军事系统，包括军队维持费、赏赐。


#### 资源三：名望（社会影响力）

名望代表了玩家角色在天下人眼中的个人威信与影响力，是消耗类资源。

**来源**：主要来源是军事系统、官职系统、活动系统、谋略系统、派系系统。包括赢得战争、官职带来的每月名望、举办科举、修缮孔庙、大型工程等活动、在谋略系统中挑战他人地位等政治计谋、在派系系统中取得政治斗争的胜利等。

**消耗**：主要消耗在军事系统、角色交互，包括发动战争、战败；名望作为社会影响力可以在多个系统中作为交互消耗类资源，如在官职系统中，求取任命更高等级的职位；说服其他角色加入；领地系统中推动某些改革。

#### 资源四：合法性（统治权威）

合法性代表了玩家统治的"正当性"——百姓和下属是否认为你有资格统治他们。这是最具中国特色的资源。

**来源**：主要来源是官职系统。包括获得朝廷正式任命的官职（最重要的来源）；击败叛军（"奉诏讨逆"后合法性大增）；举行正式的就职仪式；宗教（佛道）背书、民心向背。

**消耗与损失**：合法性不直接消耗，但会因为某些行为而下降；包括无故发动战争（"武力兼并"）；被朝廷宣布为"叛逆"；继承危机期间（继承人尚未获得朝廷认可时，合法性持续下降）。

## 7. 官职系统

我们认为，作为东方历史的模拟扮演游戏，在角色之外，官职是最重要的系统，是玩家目标感、代入感、成就感的重要来源，也是玩家目标的重要来源。无论是在官阶体系中，从一个小吏一步步成为宰相，权倾朝野；亦或是跟同僚们之间的互动；还是作为节度使，一步步扩张领地，获取更高的官职，官职系统应该在这其中扮演十分重要的角色。此前源自西方的历史模拟游戏，很难充分地模拟这一体系。而许多传统的网页游戏，例如《皇帝成长计划》等，又失之过于简单。

我们希望这个系统在权衡游戏性和正史性的基础上，给玩家带来尽可能多的成就感、代入感，让玩家体会到成为“天策上将、太尉、尚书令、陕东道大行台尚书令、益州道行台尚书令、雍州牧、蒲州都督、领十二卫大将军、中书令、上柱国、秦王”的快感。

**关联的核心资源**：钱、名望、正统性
**系统中的二级资源**：贤能

### 7.1 职位（职）与品位（官）两大体系

中国古代的官阶制度可以分为“职位”和“品位”两大概念，“职位”指的是官员在国家行政机构中所担任的具体职务和岗位，以“事”为中心，对应的是具体的行政职能、责任分工和实际权力，举例： 县令、太守、六部尚书、大理寺卿等，这些都是具体的“职位”，说明了该官员具体负责管理什么事务。“品位”指的是官员个人所拥有的级别、资格、身份和薪酬待遇的标准，以“人”为中心，代表了官员在整个官僚集团乃至社会中的地位和声望，标志着个人的资历和荣誉。举例： 魏晋的“九品”、唐代的“散阶”（如光禄大夫、建威将军）、明清的“特进荣禄大夫”以及各种爵位、勋官等。

### 7.2 官职的获取与剥夺

#### 品位的获取与剥夺
品位的获得依赖角色的贤能。贤能是一种点数，通过角色在终其一生的官场浮沉中，通过担任要职、施行善政而积累起来的。贤能不能像货币一样被花费，而是用来衡量角色与其他职位候补的排名。贤能达到一定数值，既可以获得对应的品位。
**贤能的获得**：通过科举、皇帝及高级官员的特殊授予、担任职位、治理领地、活动获得；
**贤能的降低**：御史台官员的弹劾、皇帝或高级官员的特殊剥夺等会导致角色贤能的降低；

#### 职位的获取与剥夺
- **朝廷任命**：由皇帝、吏部或上级官员任命，合法性最高；
- **军事占领后追认**：玩家占领某地后，向朝廷上报请求"追认"，需要花费一定的名望，可能导致合法性降低。
- **自立留后**：节度使死后，其子自立为"留后"，等待朝廷承认。这是晚唐最常见的权力传承方式，但合法性会暂时下降。
- **任命低级职位**：角色可以任命自己的下属职位，如节度使可以任命下级的刺史、防御使；也可以任命都虞候、节度判官等。

### 7.3 上级与下属
**法理上下级** 这些职位之间存在法理上的上级与下属关系，例如御史中丞是殿中侍御史的上级，三司使是三司判官的上级。在地方节度使所对应的路，其下的法理州对应的刺史/防御使，是节度使的法理下级。
**实际上下级** 在中央，法理上下级与实际上下级永远是一致的。例如，角色从监察御史调任三司判官，则其上级也就由御史中丞变成了三司使。在地方，由于领地有实控人，可能导致法理上级与实际上级不同。
**下属的义务** 在地方，下属需要将领地所获得的钱、粮、士兵的一部分或大部分上交给其上级。上缴比例取决于领地的集权体制。
**上下级互动** 上级可以通过发布命令交互，向下级发布指令，包括让下级在地方修建建筑、举行活动等。

## 8. 领地系统

### 8.1 领地等级
游戏的领地分为三、道、州两个等级，分别对应王、刺史、节度使。
**州**：是游戏中最基本的领地单位，拥有多个建筑槽位，可以进行建筑修建。具有控制度（Control）、发展度（Development）和民众支持度三个关键属性。为州的拥有者提供钱、粮、士兵。
**道**：是州的上级单位，由多个州构成，其中一个州会成为道的治州，也就是节度使本人及幕府所在。
**国**：是道的上级单位，由多个道构成，其中一个州会成为国的首都，也就是王本人及宫廷所在。该领地头衔仅能在危世和乱世创立。

### 8.2 领地类型
游戏中的领地分为民政和军事两个类型。民政州的长官为刺史，民政道的长官为观察使；军事州和路也被称之为军，州级长官为防御使，军事路的长官为节度使。

### 8.3 法理领地与实控领地
法理领地是指历史传统或法律上公认的领属关系，每个州在法理上都固定从属于一个特定的路，而每个路则固定从属于一个特定的国。所有国都共同归属于“天下”的法理之下。
**法理宣战** 当我们拥有一个高级头衔（如道）时，如果该头衔法理下的某些州被其他势力控制，我们就会自动获得收复这些领地的“法理宣战理由”，发动战争将更加容易。反之，如果我们向非法理的领地发起战争，则会更加困难。
**职位的获取** 当我们实际控制某个高级头衔的51%以上的下级头衔时，我们可以要求成为这个高级头衔的主官，消耗名望、钱。
**法理转移** 可以花费名望要求皇帝（或本身就作为皇帝），进行法理的重新规划。

### 8.4 领地体制
领地体制（Realm Laws）是我们管理领地内部秩序、控制下属以及确保权力平稳交接的核心机制。
**集权体制** 决定了我们对领地和下属的控制力，分为四个等级，等级越高，我们拥有的特权越多，但面临的下属好感度惩罚也越重。一级集权：我们只能获得常规的钱粮士兵收入，下属可以自由地相互宣战或对外扩张。
二级集权：我们可以剥夺下属的头衔，以及更改领地的继承法。三级集权：下属除非对我们拥有牵制，否则无法对领地内的其他下属宣战。四级集权：下属完全被禁止发动任何战争，我们可以进行继承人的指定。
**继承体制** 决定了我们以何种方式进行领地的继承。
**边境战争体制** 决定了边境的道是否有自行发动战争的权利，以及皇帝是否会第一时间参与战争。
**致仕年龄** 决定了领地内的官员什么年龄退休致仕。
**部队体制** 决定了军制，包括是否允许下级拥有牙兵、是否允许下级拥有官健等。

## 9. 军事系统

### 9.1 战争理由

战争不能随意发动。发动战争需要一个"战争理由"，。

**战争理由的核心原则** 合法性作为基础，需要消耗一定的名望，可能花费金钱，并造成一定的合法性变化。

**战争理由的类型**

1. 武力兼并：在任何情况下（除非四级集权）都可以向相邻的统治者使用该理由，在治世将消耗巨量的名望，造成大量合法性降低；在危世使用将导致合法性受损，消耗大量名望；在乱世该理由将转变为逐鹿天下，无合法性损失，名望消耗减少

2. 宣称类：包括个人宣称 (Personal Claims)，通常通过伪造宣称、家族继承或皇帝命令获得。法理宣称 (De Jure Claims)： 如果我们拥有一个高级头衔（如节度使），我们自动拥有向占领该法理下属领地（如州）的其他藩镇宣战的理由。为其他人宣称 (Pushing Claims for others)： 我们可以利用宫廷或幕府中其他人的个人宣称对外宣战。如果胜利，该廷臣将获得土地；如果该土地级别低于我们的主头衔，他将成为我们的下属。

3. 时代类：在危世，有特殊的宣战理由包括：奉召讨逆、矫诏征伐、独立；在治世，有特殊的宣战理由包括：开疆拓土。

### 9.2 军队系统

#### 军队类型
游戏中有四种军队，各有优劣：
1. 牙兵：最精锐的部队，战斗力很强，自成体系，但士气下降快，需要通过持续的赏赐、战斗胜利等进行维持。士气过低，则牙兵可能会拒绝行动或发生叛乱。
2. 官健：府兵，依附于领地，战斗力较强。晚唐时期，府兵制度已经败坏。
3. 乡兵：乡兵，依附于领地，战斗力很弱。
4. 藩兵：雇佣兵，战斗力较强，仅在边境地区能够雇佣。

#### 兵种及兵种克制
游戏中的牙兵及官健拥有不同的兵种，包括：
1. 重步兵
2. 重骑兵
3. 轻骑兵
4. 轻步兵
5. 长枪兵
6. 弓箭手
它们之间拥有不同的克制关系、地形修正，以及属性数值。

#### 军队属性
每一支军队都拥有以下属性：数量、伤害、生命值、士气、行军速度、攻城值。

#### 军队将领
每一支军队都需要一位将领，将领的军事能力及特质会影响军队的战斗力。

#### 军队的创建
1. 牙兵：花费钱粮进行创建，拥有直接效忠对象，不依托于领地。
2. 官健：花费钱粮进行创建，依托于领地。
3. 乡兵：不需要创建，数量由领地决定，战争期间可以直接集结。
4. 藩兵：雇佣兵。

### 9.3 战争的推进机制

核心思想是围绕补给、水道、关隘进行作战。

**第一步：宣战与集结**
- 玩家选择战争理由，消耗名望宣战；
- 集结军队。

**第二步：行军与后勤**（持续进行）
- 军队在地图上移动，每进入一个新的州，需要建立"补给线"；
- 补给线越长，补给效率越低；补给效率低于 50% 时，军队士气开始下降。
- 存在水道，会有利于补给效率；
- 在特定的州，可以修建关隘建筑，军队无法直接通过该州或该州相邻的州，必须攻占该州后才能继续推进。

**第三步：关键战役**（当双方主力相遇时触发）
- 系统弹出"关键战役"面板，玩家选择战术进行作战。

**第四步：围城与攻克**
- 攻克城池需要时间（根据城防等级，从 1 个月到 12 个月不等）；
- 可以使用攻城器械加快进度（消耗钱粮建造）；

### 9.4 战争的胜利机制

关键战役的获胜及围城攻克都会获得胜利分数，达到 100% 胜利分数即可要求对方强制投降。

## 14. AI 史书生成管线

这是本游戏的核心技术创新。程序团队需要构建一个三层数据管线。

### 14.1 设计理念

史书是游戏的终极产出，不是附加功能。整个事件系统的设计都服务于最终的史书质量。

核心原则直接继承自你已有的"史书撰写 Prompt"：
- **词句锤炼**：单字动词，四字短语，禁现代白话
- **叙事骨相**：事实白描，恪守史官中立
- **体例规制**：干支纪年，传主主次分明
- **游戏机制转化**：数值暴增变"附者如云"，拉拢送礼变"厚赂"

### 14.2 三层处理流程

**第一层：事件记录（每个行动执行后立即写入）**

每个有意义的游戏行为生成一条结构化 `GameEvent`，包含：发生时间（精确到日）、事件类型、涉及人物、涉及领地、结构化结果描述、重要性等级（1-3）。

结果描述使用半文言模板，而非纯白话。例如：
- ✗ "李克用带着3000骑兵打败了黄巢的20000人"
- ✓ "李克用以代北骑三千大破贼众两万于汴水，斩首三千级，贼帅黄巢退保陈州"

这一层的输出质量直接决定最终史书质量。

**第二层：年度筛选与汇总（每年腊月）**

将全年事件按重要性筛选，构建 AI 调用的上下文：
- 重要性 3（重大事件）：全部保留，详细描述
- 重要性 2（一般事件）：保留，简要描述
- 重要性 1（次要事件）：仅在篇幅允许时保留

**第三层：AI 文言转化**

将筛选后的事件列表发送给 AI 模型，附带系统 Prompt（基于你已有的"史书撰写"规范），生成正式的编年体文言文。AI 可以：
- 压缩冗余事件为一句话
- 为重大事件添加虚构史源注释（如"《唐末会要》曰"）
- 保持全年叙事的连贯性和节奏

AI 不可以：
- 编造未发生的事件
- 添加主观评价（除"史官按语"外）
- 使用现代白话

### 14.3 AI 后端设计

采用 `ChronicleProvider` 抽象层，支持多模型后端切换：

```
ChronicleProvider (接口)
  ├── MoonshotProvider  (Kimi K2.5)
  ├── AnthropicProvider (Claude)
  └── LocalProvider     (离线占位，返回原始事件列表)
```

缓存策略：已生成的年度史书存入 IndexedDB，不重复调用。
