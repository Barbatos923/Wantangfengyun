# 晚唐风云计谋系统设计

## 1. 文档目标

本文基于 CK3 的 `scheme` 骨架，结合《晚唐风云》现有的角色、关系、官职、合法性、互动与 AI 模块，设计一版可落地的计谋系统。

这套系统的目标不是简单复刻 CK3 的“谋杀/拉拢”，而是把晚唐政治生态中的：

- 朝廷倾轧
- 宦官干政
- 藩镇离合
- 台谏清议
- 幕府运作
- 军府密谋

统一进同一套可扩展框架。

## 2. 设计目标

- 让“关系玩法”和“权力玩法”共享同一套语言。
- 让计谋既能服务剧情，也能稳定进入 AI 决策。
- 让人物性格、能力、官职、地位、关系、资源都能影响计谋，而不是只看一个数值。
- 让朝廷与藩镇的政治结构天然成为计谋网络的一部分。
- 让系统支持“轻量计谋”和“重量计谋”两层复杂度，避免每个计谋都做成独立小游戏。

## 3. 参考 CK3 后保留的骨架

CK3 里最值得保留的不是具体计谋，而是这条公共流程：

`发起条件 -> 目标选择 -> 计谋配置 -> 周期推进 -> 中途事件 -> 暴露/反制 -> 结算 -> 后果`

本文保留以下 CK3 核心思想：

- 计谋按模板定义，不按事件硬编码。
- 推进速度与最终成功率分离。
- 失败与暴露分离。
- 轻量计谋不必带代理人与机会点。
- 重量计谋可以积累“准备度”和“机会点”，并在关键时刻进入执行。
- 目标一侧可以长期部署反制措施。
- AI 需要理解计谋的收益、风险、投入与人格适配。

## 4. 与现有项目的结合点

这套设计优先对接当前仓库已有系统：

- `src/engine/character/types.ts`
  角色已有能力、特质、关系、资源、官职、所属与位置信息。
- `src/data/traits.ts`
  已有较完整的人格维度，适合用于计谋偏好、代理人适性、风险容忍度。
- `src/engine/systems/socialSystem.ts`
  已有关系衰减、德行、合法性、声望增长逻辑，适合承接计谋后果。
- `src/engine/interaction`
  适合作为玩家发起计谋的入口层。
- `src/engine/npc`
  适合作为 AI 发起、终止、追加投入、执行结算的驱动层。
- `src/engine/official`
  适合作为“官职网络、机构权限、合法性、任免关系”的规则来源。

## 5. 系统定位

《晚唐风云》的计谋系统不是“单纯阴谋系统”，而是“政治行动系统”的一部分。

按玩法层级，分成两类：

- 基础计谋：轻量、周期短、无代理人或少量代理人、主要服务关系塑造与软控制。
- 重大计谋：周期长、可投入门路与代理人、存在反制与暴露风险、会改变权力结构。

按政治空间，分成三类：

- 宫廷计谋
- 藩镇计谋
- 跨势力计谋

按目的，分成五类：

- 关系类
- 情报类
- 控制类
- 破坏类
- 权力类

## 6. 核心循环

### 6.1 玩家视角

1. 选择目标与计谋。
2. 查看基础成功、隐秘、周期、预计收益、预计风险。
3. 决定是否追加投入。
4. 选择门路、代理人、资源倾向。
5. 计谋按月推进。
6. 触发中途事件与关键抉择。
7. 在合适时机执行，或被迫暴露、夭折、自动结束。
8. 结算收益、关系变化、官场风评、合法性与后续连锁。

### 6.2 系统视角

1. 互动入口只判断是否可以发起。
2. 计谋对象实例化后进入全局 `SchemeStore`。
3. 每月由 `socialSystem` 或新增 `schemeSystem` 统一推进。
4. `npc` 根据人格、利益和风险决定是否继续投入、撤回或执行。
5. `storyEventBus` 用于派发计谋事件。
6. 结果回写角色关系、资源、官职状态、地块控制与合法性。

## 7. 核心概念

### 7.1 计谋模板 Scheme Template

计谋模板是静态定义，决定：

- 名称
- 分类
- 目标类型
- 发起条件
- 基础周期
- 基础成功率
- 基础隐秘度
- 可用代理人类型
- 可用投入项
- 关键事件池
- 成功结果
- 失败结果
- 暴露结果

### 7.2 计谋实例 Scheme Instance

计谋实例是运行中对象，记录：

- 发起者
- 目标
- 当前阶段
- 当前进度
- 当前成功率
- 当前隐秘度
- 当前风险值
- 当前门路值
- 已投入资源
- 已加入代理人
- 已发生的事件标记
- 是否已暴露
- 是否进入执行窗口

### 7.3 轻量计谋

轻量计谋借鉴 CK3 的 `is_basic = yes` 思路：

- 没有复杂代理人槽位
- 没有机会点上限
- 一般不进入关键筹备界面
- 每月推进
- 到达结点直接掷结果

适用示例：

- 结交
- 笼络
- 试探
- 挑拨
- 散布风闻
- 私下通款

### 7.4 重量计谋

重量计谋借鉴 CK3 的谋杀、捏造把柄、夺位框架：

- 可以配置代理人
- 可以投入资源换取“门路”或“准备度”
- 有暴露判定
- 可以被反制
- 有关键时刻和执行窗口

适用示例：

- 构陷宰相
- 联络神策军废立
- 煽动牙将兵变
- 收买台谏弹劾
- 图谋夺镇
- 密杀节度使

## 8. 计谋的五项核心数值

建议每个计谋实例至少维护以下五项主数值：

- 进度 `progress`
  表示距离进入结算窗口还有多远。
- 成功 `success`
  表示一旦执行，达成目标的概率。
- 隐秘 `secrecy`
  表示在推进过程中不被察觉的能力。
- 风险 `risk`
  表示一旦出事会造成多大反噬。
- 门路 `access`
  表示你在目标所在政治网络中可调用的暗线、熟人、通路与可操作空间。

### 8.1 为什么增加“门路”

CK3 的机会点适合宫廷阴谋，但晚唐更强调“人脉通路”和“能否伸手进去”。

因此建议把 CK3 的 `opportunity` 本地化成“门路”，它代表：

- 你是否有可接触的中官、书吏、牙将、幕僚、驿卒
- 你是否已掌握把柄、书信、账簿、军令链
- 你是否具备让行动落地的实际渠道

门路既是资源，也是执行门槛。

## 9. 数值来源

### 9.1 发起者能力

按计谋类型读取发起者主能力：

- 关系类主要取 `diplomacy`
- 情报类主要取 `strategy`
- 文书法理类主要取 `scholarship`
- 财计运作类主要取 `administration`
- 兵变与军事渗透类主要取 `military + strategy`

### 9.2 目标抗性

目标抗性来自：

- 自身能力
- 官位层级
- 是否在主场
- 是否有强势上司或保护人
- 是否掌握机构资源
- 是否被重臣、禁军、藩兵、谏官保护

### 9.3 性格修正

性格不是简单加减成功率，而应影响不同环节：

- `boldness`
  更愿意发起高风险计谋，也更愿在关键时刻提前执行。
- `compassion`
  压低构陷、刺杀、清洗类计谋倾向。
- `greed`
  提高收买、勒索、夺利类计谋倾向。
- `honor`
  压低背盟、伪诏、暗杀类计谋倾向。
- `rationality`
  提高长期筹备倾向，减少冲动执行。
- `sociability`
  提高关系类计谋成功，也提高拉拢代理人能力。
- `vengefulness`
  强化报复型计谋权重。
- `energy`
  提高可同时维护计谋数与追加投入频率。

## 10. 计谋分类设计

### 10.1 关系类

用途：

- 提升好感
- 降低敌意
- 软化藩镇关系
- 建立私交网络

示例：

- 结交
- 笼络
- 送礼结欢
- 私荐子弟
- 代为周旋

特点：

- 多为轻量计谋
- 不强调隐秘
- 结果主要回写 `relationships`

### 10.2 情报类

用途：

- 获取把柄
- 发现账簿
- 掌握密信
- 侦知兵变苗头
- 查知结党网络

示例：

- 探访幕府
- 刺探中枢
- 搜访秘事
- 追查军饷

特点：

- 以 `success + secrecy` 为核心
- 成功后常生成“情报资产”
- 为其他重量计谋提供前置条件

### 10.3 控制类

用途：

- 操控任命
- 影响审核
- 诱导弹劾
- 干扰调兵
- 影响诏令下达

示例：

- 收买中使
- 笼络翰林
- 串联台谏
- 渗透三司

特点：

- 与机构、官职、朝廷网络强绑定
- 常改变流程结果，而不一定直接伤害目标

### 10.4 破坏类

用途：

- 制造丑闻
- 伪造文书
- 离间主从
- 破坏名望
- 削弱统治基础

示例：

- 构陷
- 诬告
- 挑拨牙将
- 散播流言
- 断其军饷门路

特点：

- 暴露后反噬大
- 常附带合法性、名望与关系链伤害

### 10.5 权力类

用途：

- 夺镇
- 废立
- 逼迁
- 清洗中枢
- 扶植代理人

示例：

- 图谋夺镇
- 联络禁军废立
- 密谋驱逐监军
- 扶立傀儡节帅

特点：

- 周期长
- 代理人要求高
- 一旦成功直接改变权力格局

## 11. 晚唐特色的代理人系统

CK3 的代理人本质上是“提供不同贡献类型的参与者”。晚唐应改写为“可穿透不同权力网络的节点”。

建议代理人不按职业通用命名，而按晚唐生态命名：

- 中官
- 枢密院吏
- 翰林学士
- 给事中/补阙
- 御史
- 三司判官
- 节度判官
- 掌书记
- 牙将
- 都虞候
- 神策军中层军官
- 僧道
- 商贾
- 驿卒
- 家奴/近侍
- 宗亲
- 外戚

### 11.1 代理人贡献类型

每个代理人只需对以下贡献类型中的一到两项特别强：

- 成功加值
- 成功上限
- 进度速度
- 隐秘加值
- 风险缓解
- 门路生成
- 暴露转嫁

### 11.2 代理人适配逻辑

代理人加入意愿建议由以下因素共同决定：

- 与发起者关系
- 与目标关系
- 性格相容性
- 是否同机构
- 是否同阵营
- 是否能从结果中获利
- 是否能被钱粮、人情、升迁、赦免、封赏收买

### 11.3 代理人不是永久追随者

需要支持：

- 中途退出
- 倒向目标
- 被策反
- 暴露后供出主谋
- 临阵退缩

这会让计谋有更多中间状态，而不是纯概率条。

## 12. 反制系统

CK3 里反制是目标方长期部署的防御措施。晚唐可以直接做成“制度性防谍”与“临时戒备”。

建议分成三类：

- 常设反制
- 机构反制
- 应急反制

### 12.1 常设反制

示例：

- 宫禁整肃
- 严核诏令
- 加强门禁
- 整饬驿传
- 清查幕府

效果：

- 长期提高隐秘检定难度
- 提高代理人拉拢成本
- 降低门路生成效率

### 12.2 机构反制

示例：

- 御史台巡按
- 枢密院稽核
- 三司查账
- 神策军内查
- 藩镇牙兵清点

效果：

- 针对特定计谋类别
- 对控制类、情报类、破坏类分别克制

### 12.3 应急反制

示例：

- 临时封门
- 扣押文书
- 先发弹劾
- 紧急换将
- 夜间搜捕

效果：

- 短期高强度反制
- 代价高
- 常由 AI 在察觉危险后触发

## 13. 暴露系统

晚唐计谋不能只有“成”或“败”，需要有以下暴露层级：

- 线索暴露
- 代理人暴露
- 计谋暴露
- 主谋暴露
- 伪证被揭穿

### 13.1 暴露后的常见后果

- 关系恶化
- 官场声望受损
- 合法性下降
- 被弹劾
- 被外放
- 被罢免
- 被削权
- 被围攻
- 给目标提供反击理由

### 13.2 暴露不是必死惩罚

需要支持：

- 把责任推给代理人
- 用更大代价压下去
- 以合法程序反咬一口
- 强行提前执行

这样“暴露”会变成二次博弈，而不是流程终点。

## 14. 计谋结果类型

建议把结算标准化为以下结果接口：

- 关系变化
- 资源变化
- 官职变化
- 机构控制变化
- 领地控制变化
- 合法性变化
- 名望变化
- 新状态添加
- 新情报资产添加
- 新仇恨链生成

## 15. 计谋与现有系统的耦合方式

### 15.1 与 `interaction` 的耦合

新增计谋入口时，不必一次做复杂 UI。

第一阶段可沿用现有互动模式：

- 菜单里显示可发起计谋
- 点开后进入参数面板
- 参数面板只做目标、投入、执行倾向选择

建议新增：

- `src/engine/interaction/startSchemeAction.ts`
- `src/engine/interaction/cancelSchemeAction.ts`
- `src/engine/interaction/commitSchemeAction.ts`

### 15.2 与 `npc` 的耦合

AI 需要回答四个问题：

- 值不值得开这个计谋
- 值不值得继续投资源
- 是否该在当前节点执行
- 暴露后是否止损或转向

建议新增一组行为：

- `startSchemeBehavior`
- `supportSchemeBehavior`
- `executeSchemeBehavior`
- `coverSchemeBehavior`
- `counterSchemeBehavior`

### 15.3 与 `socialSystem` 的耦合

计谋适合作为每月社会系统的一部分推进：

- 推进进度
- 刷新关系修正
- 结算轻量计谋
- 触发暴露检测

### 15.4 与 `official` 的耦合

权力类计谋需要依赖：

- 机构归属
- 官职高低
- 是否直接接触诏令、军令、财政、谏议链
- 是否具备合法程序入口

例如：

- 收买翰林适合伪造诏意
- 收买御史适合发起弹劾
- 收买节度判官适合控制幕府行政链
- 收买牙将适合撬动兵变

### 15.5 与 `legitimacy` 的耦合

计谋不应只改关系，还应改“统治叙事”：

- 公开合法动作成功，合法性上升
- 阴谋败露，合法性下降
- 诬陷无果，名望与合法性双损
- 借台谏、法统完成清洗，则损失较小

## 17. 首批推荐计谋

为了尽快验证系统，建议先做 6 个，覆盖轻量与重量两层。

### 17.1 轻量计谋

- 结交
  提升目标好感，适合作为关系系统延伸。
- 挑拨
  降低目标与第三人的关系。
- 探访秘事
  获取目标弱点或情报资产。

### 17.2 重量计谋

- 构陷
  通过伪证、台谏、账目、证人制造政治打击。
- 联络牙将
  在藩镇内部积累门路，为兵变或夺镇铺路。
- 收买中使
  在宫廷链条中插入自己的通道，影响诏命、传旨与情报。

## 18. 推荐的三条玩法链

### 18.1 宫廷链

`结交翰林 -> 探访秘事 -> 收买中使 -> 构陷政敌 -> 发动弹劾`

### 18.2 藩镇链

`结交幕僚 -> 联络牙将 -> 控制军令 -> 图谋夺镇`

### 18.3 朝藩联动链

`朝中构陷节帅 -> 地方挑拨牙将 -> 借机换镇/削权`

## 19. AI 设计原则

AI 不应无脑追求最高成功率，而应按人格与身份分流：

- 高胆识、高复仇、低荣誉人物更倾向破坏类与权力类计谋。
- 高社交、高理性人物更倾向关系类与控制类计谋。
- 地方强藩更倾向藩镇链条。
- 朝廷重臣更倾向宫廷链条。
- 合法性低的统治者更倾向先做控制类和关系类计谋稳盘，而不是直接清洗。

AI 每次评估时建议比较：

- 预期收益
- 成功率
- 暴露概率
- 暴露后损失
- 当前资源承受能力
- 是否存在更低风险替代行动

## 22. 设计上的几个关键取舍

- 不建议一开始就做 CK3 那种非常重的“事件海”，先把数值闭环和政治后果跑通。
- 不建议把所有计谋都做成“谋杀模板换皮”，晚唐更适合“情报-控制-构陷-夺权”的连续链。
- 不建议只围绕皇帝设计计谋，藩镇、幕府、军府内部斗争应该同样重要。
- 不建议让隐秘度完全决定一切，晚唐政治里“半公开施压”“借法统斗争”同样重要。

## 23. 一句话总结

《晚唐风云》的计谋系统，应当是一个以“门路、关系、机构、法统、兵权”为核心的政治行动框架；它借鉴 CK3 的公共骨架，但最终服务的是晚唐特有的朝廷与藩镇双层博弈。
